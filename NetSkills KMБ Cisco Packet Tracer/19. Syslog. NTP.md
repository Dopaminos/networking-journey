Логи - невероятно важный аспект администрирования, и даже не важно чего: Серверов, Контейнетр, Сетей, Баз данных.

Логи - данные, с помощью которых можно отследить и устранить любые проблемы. У нас есть несколько способов получить логи:

 - Console logging 
 - Buffered logging (RAM)
 - Terminal logging (terminal monitor)
 - Syslog
 - SNMP Traps
 - AAA Accounting 


У этих сообщений есть уровни:

0. Emergencies
1. Alerts
2. Critical
3. Errors
4. Warning
5. Notification
6. Informational
7. Debugging

В этом уроке будут рассмотрен протокол Syslog, работающий по UDP, port 514.

Приступим к практике. 
![](./Картинки/SYSLOG1.png)
Построим простую сеть с 2 ПК, свичом, сервером и роутером.

Рассмотрим первый способ логгирования, а именно Console logging. Зайдём в CLI Switch-a, здесь, с помощью команды 
 - `logging console` можно изменить его уровень (0-7), 
 - `logging on/off` вкл/выкл логи, 
 - `logging buffered <byte size>` размер буфера
 - `logging trap` задаёт уровень логгирования
 - `logging <ip>` логгирует всё, что происходит на консоли `<ip>`
   
`ser t l d ms` - меняет отображение времени в syslog сервисе.

NTP - протокол для синхронизации времени, работающий на UDP 123.
Соответствующий сервер можно настроить на роутере с помощью команды `ntp server ip`.
