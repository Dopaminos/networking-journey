Trivial File Transport Protocol. Используется для загрузки бездисковых рабочих станций, обновлений прошивок и файлов конфигураций.

Типы конфигураций Cisco:
 - runnig-confing (текущая, хранится в рам)
 - startup-config (при запуске, хранится в флеш или хдд)

TFTP серверов куча, выбирай не хочу. Приступим к практике.

Возьмём сеть из предыдущего урока, ничего менять не будем, т.к. там так же будут настройки сверху. В CPT прошивки грузятся из каталога `flash:/`, содержимое которого можно посмотреть с помощью `show flash

Откроем сервак и увидим, что на нём есть две версии прошивки для нашего свича (2960), попробуем обновить.

Чтобы это сделать, нужно знать, куда отправлять данные, ибо сейчас у свича нет адреса. Зададим его на VLAN1.

Далее, с помощью `сopy <source> <dest>` `ip, filename` копируем прошивку на свич и `boot system <path>` + reload = новая прошивка на свиче.

`copy startup-config <tftp>` загружает конфигурацию на сервер.
