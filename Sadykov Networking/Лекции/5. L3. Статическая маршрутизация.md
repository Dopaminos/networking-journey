Маршрутизация - процесс определения пути доставки пакета между сетями, работает на L3. Работа осуществляется с Routing Table (ip r), хопов и выбора интерфейса.

# Принцип работы маршрутизации

Для маршрутизации существует соответствующее устройство, называемое Router. 

Роутер не всегда может направить пакет напрямую к Destination IP, поэтому ему нужно определить ближайшую к нему точку, чтобы направить пакет туда и дать ему ход дальше.

Оно работает так:
- Роутер принимает пакет
- Смотрит DESTINATION_IP 
- Проверяет `ip r`
- Отправляет пакет в Default Gateway, прописанный здесь, чтобы пакет шел дальше в next hop по принципу Longest prefix match или напрямую, если DESTINATION_IP находится внутри сегмента отправителя.

Принцип Longest prefix match заключается в том, что роутером сравнивается IP-адрес назначения и записи из `ip r` с самой длинной маской подсети. Чем больше маска - тем конкретнее маршрут.

___
## Типы маршрутов
- Connected - автоматический, соответствует интерфейсу
- Static - вручную заданный маршрут (`ip r a .. via ..`)
- Dynamic - постоянно меняющийся маршрут в зависимости от доступности его узлов
Так же существует Default маршрут, который работает, если сеть неизвестна.

# Построение супер-сетей 

Супер-сеть - сеть созданная с помощью агрегированных мелких сетей. 

Построение супер-сетей обеспечивает стабильность доступа к сети благодаря уменьшению маршрутизируемого трафика, оверхеда в виде кучи хопов на множестве IP адресов мелких сетей и таблицы маршрутизации.

Например, вместо того, чтобы создавать кучу сетей вроде 172.16.0.0/24, 172.16.1.0/24, 172.16.2.0/24 и 172.16.3.0/24, роутеру можно маршрутизировать лишь 172.16.0.0/22, объединив вышеописанное в супер-сеть.
