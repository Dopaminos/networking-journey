# OSPF - Open Shortest Path First
Это более современный протокол для динамической маршрутизации, созданный для больший сетей предприятий, позволяющий максимально эффективно выбирать маршруты.

Работает OSPF по механизму Link-State, где, в отличие от RIP, роутер может увидеть топологию сети и построить её карту (Link State DataBase), с помощью формирование LSA (Link State Advertisements).

# Принцип работы 

Сами маршруты строятся с помощью дерева путей по алгоритму Дейкстры: 
 - Роутер шлёт multicast-пакет (hello) один раз
 - Если параметры у Destination Host совпадают, то назначение получает Adjacency и далее, имея таблицу соседей, при построении маршрутов, роутер общается точечно, не пытаясь каждый раз флудить в бродкаст.
Благодаря такому подходу, каждый роутер обменивается с остальными Link State Information и может построить LSDB.

Маршруты существуют, но теперь нужно выбрать лучший из них для передачи данных.
В OSPF есть метрика Cost = Reference Bandwidth / Interface Bandwidth. Чем меньше Cost - тем быстрее адресату дойти до получателя.

Для определения, кто есть кто в сети существуют Router ID, который задаётся либо самым высоким IP интерфейса, либо loopback-ом. Если интерфейс упал - ID может повредиться и мешать работе роутера.

Для управления трафиком, в OSPF сетях существуют такие роли, как Designated Router и Backup Designated Router. Это "магистральные" роутеры, которые должны распространять LSA по сети, не создавая мэш-сеть. 
Роли выбираются на основе самого высоко-приоритетного интерфейса. Если приоритеты у нескольких роутеров одинаковы - проверяется Router ID. У кого IP выше, тот и DR.
## Типы пакетов OSFP
- Hello
- DataBase Description
- Link State Advertisement
- LSU / LSR / LSAck
## Основные типы LSA
 - intra-area
 - inter-area
 - external (e1/e2)
E1 учитывает весь путь к источнику при построении маршрута,
E2 учитывает только cost внешней сети
 
Внутри OSPF сетей, обычно существуют зоны, помогающие балансировать нагрузку на сеть и не раздувать LSDB. Самой главной зоной можно назвать Area 0 (backbone), т.к. к ней подключаются все остальные зоны.

## Типы роутеров

- internal router (работает внутри одной зоны)
- area border router (работает между несколькими зонами)
- autonomous system boundary router (роутер импортирующий маршруты)
Для экономии ресурсов, между зонами передаются лишь маршруты, внешним роутерами недоступна вся топология сети, ведь она передаётся лишь внутри одной зоны.

## OSPF vs RIP

RIP изначально предназначался для мелких сетей и в целом очень ограничен и медленнен (работа до 15 хопов, единственная метрика - их число).

OSPF не важны хопы, для определения самого оптимального маршрута, ему важна лишь скорость канала считываемая как Cost. +есть поддержка CIDR, позволяя работать с разными масками подсетей.
Имеет более быстрый convergence, лучше масштабируется и строит более оптимальные маршруты
