# L3 - Сетевой уровень (Network layer)

Если L2 отвечает за доставку данных внутри одной сети, то L3 отвечает за доставку данных **между сетями** (адресация, определение маршрута отправки пакета)

Работает L3 с IP протоколом, а именно с пакетами IPv4/IPv6.

IP-адрес - адрес, содержащий 32 бита данных при IPv4, поделённый в октеты.

Любое устройство, имеющее IP может маршрутизироваться, формировать IP пакеты и т.д.

# IP (Internet Protocol)

```
IP - протокол доставки пакетов без гарантий
```

Функция этого протокола ограничена доставкой, никаких проверок, внутри этого протокола, не имеется, что можно назвать `best effort delivery`.

Надёжность пакетов обеспечивает протокол 4-ого уровня, а именно TCP. О нём позже.

## Структура IP пакета

Пакет состоит из двух частей: хэдер (заголовок) и данные.

Заголовок состоит из:
- source+destination ip
- TTL
- Протокол
- Длина (регулируется MTU)
- Флаги фрагментации
Здесь вводится новые для нас термины, а именно TTL и MTU.

TTL (time to live) - время жизни пакета в сети или же счётчик хопов. 
MTU (max. transmission unit) - максимальный размер пакета в байтах, который можно передать без фрагментации

# Router

Роутер (маршрутизатор) - устройство, работающее сугубо на L3, маршрутизирующее пакеты IP.

## Принцип работы 

У каждого роутера есть своя таблица маршрутизации трафика, 
например, `ip route` в пакете iproute2 Линукса.
Роутер уменьшает TTL каждый хоп, что обеспечивает защиту от петель.

При отправке пакеты куда-то, роутер ищет маршрут с помощью этой таблицы. Если в таблице нет нужного адреса, пакет отправляется к основному шлюзу (Default Gateway) и его маку, что есть следующая точка пути пакета.

# IP-адресация

Адреса в IP разбиваются на сетевую и хостовую часть. Разбитие адреса работает с помощью маски подсети.

## CIDR (бесклассовая адресация)

Бесклассовая адресация позволяет легко разделять сети по адресам, например /24, /25, /21..
Число после слэша обозначает кол-во единичных битов, занятых сетевой частью IP-адреса.

Например - /24 означает, что маска подсети = 255.255.255.0, и хостовая часть адреса равна всего лишь восьми битам.

# Подсети (VLAN)

Разделение сети предназначено для изоляции трафика, более гибкого управления им и уменьшения broadcast-запроса.

Например, сеть `192.168.1.0/24` можно разделить на 
```
192.168.1.0/25
192.168.1.128/25
```

## Широковещательный домен.

Это часть сети где broadcast достигает всех устройств, разделяемая роутером.

## Типы портов

- Access port
Порт, принадлежащий лишь одному VLAN-у, предназначен для получения устройствами доступа к общей сети через L2 устройства.

- Trunk (магистральный) port
Порт, который связывает несколько VLAN-ов для передачи трафика между ними.

Трафик, передаваемый через транк порты имеет тег, внедряемый в заголовок Ethernet Frame по стандарту IEEE 802.1Q. 

Структура тега:
- Заголовок MAC-а
- VLAN ID
- Приоритет трафика в сети
- Полезная нагрузка

## Sub-интерфейсы

Такие подинтерфейсы нужны для конфигурации вида "роутер на палочке" (hi int17h). 

Конкретнее говоря, L2 коммутаторы не могут маршрутизировать трафик между VLAN-ами, например 192.168.1.x и 192.168.2.x, чему очень помогает роутер, подключенный к коммутатору. Для этого понадобится лишь один Cat кабель, и подинтерфейсы позволят маршрутизировать весь трафик 

Такая конфигурация так же называется Inter-VLAN Routing, но она не всегда нужна, т.к. современные коммутаторы так же имеют механизмы работы на L3.

## DHCP в L3

Поскольку DHCP работает с помощью Broadcast-ов, он не проходит через роутеры и для корректной работы технологии используется DHCP relay, который позволяет роутеру принять DHCP Discover и пересылает его серверу
