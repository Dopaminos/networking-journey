Есть протоколы работающие с внутренней сетью, есть протоколы работающие с внешней сетью, поэтому не всегда можно сказать, что одно лучше другого, у всего есть свои задачи.

BGP (Border Gateway Protocol) - path-vector протокол междоменной маршрутизациий для обмена маршрутами. 
Он работает между автономными системами вроде провайдера, крупных компаний и датацентров не строя топологию как OSPF.

Протокол работает поверх TCP на 179 порте, осуществляя point-to-point соединение. Для его работы обязательна стабильная TCP-сессия.


# Подвиды BGP

## eBGP

Протокол внешнего взаимодействия между AS, используется в интернете.

## iBGP

Протокол внутреннего взаимодействия внутри одной AS. 

Имеет в себе проблему масштабирования, ввиду того, что iBGP не пересылает маршруты дальше, он лишь получает маршруты от других.

Её решением часто выступает Route Reflector, который отдаёт маршрут клиентам. В Cisco это ручная опция, в Mikrotik включено автоматически.

# Принцип работы

Маршруты, по протоколам BGP, строятся по принципу приоритетов:
- next-hop reachability
- weight (cisco) 
- local preferece
- shortest AS path
- origin
- MED (Multi-Exit Discriminator)
- eBGP > iBGP
- shortest next-hop
- router-id
  
Всё это - локальные параметры, где "чем больше тем лучше" и именно по ним происходит создание более оптимального маршрута. 

## Типы маршрутов

- network (вручную объявленные сети)
- learned routes (сети от соседей)
- redistributed (из других протоколов)
# Межпротокольное взаимодействие

Сети, работающие по разным протоколам, можно объединить в единую с помощью команды `redistribute`, однако при таком взаимодействии протоколов есть свои недостатки, например:
 - Несовместимость метрик между собой
(например, OSPF считает Cost с помощью скорости, а RIP считает hop-ы и строит по ним маршруты)
 - Ручное прописывание некоторых метрик (следует из первого)
 - Петли при маршрутизации (OSPF сеть ведёт в RIP, RIP сеть ведёт обратно в OSPF)
