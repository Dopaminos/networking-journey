# STP и его виды

Spanning Tree Protocol - протокол, позволяющий трафику иметь лишь один путь, блокируя остальные доступные. 
Работает по принципу логического дерева сети.

## Принцип работы

STP выбирает главный коммутатор в сети и назначает его Root Bridge-ом.

Как именно он назначается?
У каждого свича в сети, если свой Bridge ID, обычно кореллирующий с приоритетом (32768), который состоит из MAC свича (6 байт) и поля приоритета. 

По стандарту, приоритет у каждого свича 32768, но его можно изменить с помощью

```
(config)#spanning-tree vlan ID priority VALUE
```

Коммутатор с самым низким BID побеждает и становится Root Bridge.

STP выбирает Root Port по наименьшему path cost. 
Path cost зависит от скорости линка, чем больше скорость - тем меньше цена.

## Роли портов

STP может назначать роли портам, а именно:
- Root (порт, который быстрее всего приведёт к бриджу)
- Designated (порт, передающий трафик к руту из сегмента сети)
- Blocked (порт, заблокированный автоматически, дабы не допустить появления петель)

## Состояния портов

Одновременно с ролями, у портов есть несколько состояний:
- Blocking (подключен, но не работает)
- Listening (слушает, но не прокидывает трафик)
- Learning (порт, создающий мак таблицу, **пока** не передающий трафик)
- Forwarding (работающий порт)
- Disabled (выключен специально)
___
У STP есть несколько подвидов имеющие свои назначения и стандарты.

## Rapid STP

Обычный STP меняет свои состояния при сбоях очень долго, обычно это может занять до 50 секунд. RSTP решает эту проблему и меняет состояния за 1-3 секунды. Стандартом данного подвида является 802.1w.

## Multiple STP

Этот подвид позволяет балансировать использование VLAN-ов в сети.
Работает с помощью создания нескольких экземпляров покрывающего дерева

## PVST (Per VLAN STP)
У циски есть режим, при котором STP строится на каждый VLAN

# BPDU (Bridge Protocol Data Unit)

Всем STP-топологиям надо как-то наладить общение между коммутаторами, для этого используется BPDU. Это фрейм с данными, обеспечивающими работу STP. 

Чаще всего BPDU используется для выбора Root Bridge, предотвращения образования петель и лучшей адаптивности сетей к изменениям топологии.

# VLAN-ы и их теги

На каждый фрейм можно повесить VLAN тег (.1q 4 байта), позволяющий идентифицировать VLAN отправления.


## Native VLAN
VLAN, работающий на магистральном порте и переносит в себе трафик без VLAN-тега. По стандарту, он является VLAN 1. 

Проблема которая может случиться в таком VLAN-e - перепутанные вланы, из-за чего трафик без тега будет стучаться в другой влан.

# Проблемы L2 сетей

Поскольку, мы уже знаем то, что работает с L2 и как (MAC, ARP, VLAN), нужно узнать, как можно сделать сеть надёжнее и более отказоустойчивой. 

Для этого, мы рассмотрим проблемы базовых механизмов сетей, а именно:
- broadcast storm
- network loops (петли)
- MAC flapping
- VLAN mismatch
- ARP storm

## Network Loops

Петли, внутри участка цепи, являются одной из самой опасных проблем в L2 сетях. Если между коммутаторами есть более одного пути трафика, то появляется петля.

Почему это проблематично?
Ethernet не имеет TTL, из-за чего Frame может циркулировать в сети бесконечно.

Циркулирование фрейма в сети нагружает CPU, т.к. фреймов в сети много и их кол-во увеличивается вместе с использованием сети.

Последствием перель можно так же назвать broadcast storm, повреждение MAC таблицы и дупликацию пакетов

Может случаться, если воткнуть оба конца патчкорда в один свич.

## MAC Flapping

Флаппинг МАК адресов - одно из последствия петель в сети, хоть петли и нельзя назвать единственной причиной этой проблемы.

Switch может начать видеть один MAC на разных портах внутри одного VLAN-a, из-за чего таблица маков постоянно обновляется и CPU нагружается соответственно.

Причинами можно назвать:
- петли (очевидно)
- NIC Teaming / LACP / PAgP (соединение нескольких сетевых адаптеров в один логический интерфейс с бОльшей пропускной способностью) 
- мосты виртуальных интерфейсов 
- неправильно настроенный магистральный порт

## VLAN Hopping

VLAN hopping - эксплойт в системе вланов, позволяющий атакующему хосту получать доступ к трафику других подсетей, к которому доступа у него быть не должно. 

Это работает с помощью фреймов с двойным тегом:
- тег влана хоста
- тег атакуемого влана
  
Достигая первого магистрального порта, коммутатор убирает влан хоста из фрейма и далее, с другим тегом, трафик достигает атакуемой подсети и... ACCESS GRANTED.

# Методы защиты / решения сетевых проблем

`BPDU Guard` - защита от подключения несанкционированных коммутаторов. 
Работает так: если access порт с этой функцией получает BPDU фрейм, то этот порт сразу же отключается.

`Loop Guard` - мониторит BPDU на руте и других портах. Если их поток заканчивается -  порт входит в состояние `loop-inconsistent` (блок) и не форвардит трафик

`Storm Control` - ограничивает бродкаст/мультикаст и неизвестные свичу юникасты. Если их слишком много - трафик интерфейса дропается.
